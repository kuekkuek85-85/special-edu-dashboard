# 강사용 PRD·산출물 수합 & 피드백 대시보드 — PRD (Claude Code용)

작성: 장평중학교 교사 이승엽 (연수 강사) · 빌드 도구: Claude Code (라이브 코딩) · DB: Firebase(Firestore)

---

## 0. 이 문서를 읽는 Claude Code에게

이건 연수 강사가 **수강 교사들의 PRD와 산출물 링크를 수합하고, AI+강사 협업으로 피드백을 주고, 진도를 신호등으로 실시간 관리**하는 웹 대시보드를 만들기 위한 PRD다. 한 번에 다 만들지 말고 9장의 구현 순서를 따라 단계적으로 빌드하라. 각 단계 끝에서 동작을 확인하고 다음으로 넘어간다.

---

## 1. 한 줄 요약

수강생(교사)은 학교·이름으로 자신을 식별해 **PRD와 산출물 링크를 제출**하고, 강사는 **AI 피드백을 검토·수정해 최종 발송**하며, 모든 진행 상황을 **신호등 색상의 방사형 대시보드**에서 실시간으로 본다.

## 2. 사용자 역할 (2종)

- **수강생(교사)**: 학교+이름 입력 → PRD 제출 → 산출물 링크 제출 → 피드백 확인 → 1차/개선 재제출
- **강사(이승엽)**: 전체 대시보드 조회 → AI 피드백 검토·수정·발송 → 멘토링 우선순위 판단

> 인증은 복잡한 로그인 대신 **학교명 + 이름을 직접 입력**해 식별(연수 현장용 경량 방식, 드롭다운 아님). 같은 학교+이름이면 기존 카드로 이어서 작업, 없으면 새로 생성. 강사 화면은 별도 비밀번호(또는 강사 전용 경로)로 보호.

## 3. 핵심 개념: 신호등 상태 (5단계)

각 수강생은 아래 상태 중 하나를 가지며, 대시보드에 **색상 원(신호등)**으로 표시된다.

| 색상 | 상태 코드 | 의미 | 강사 액션 |
|---|---|---|---|
| 🔴 빨강 | `RED` | 1차 PRD·링크 **둘 다 미제출** | 우선 멘토링 대상 |
| 🟠 주황 | `ORANGE` | 1차 **PRD만** 제출 (링크 없음) | 링크 제출 독려 |
| 🟡 노랑 | `YELLOW` | 1차 **PRD+링크 둘 다** 제출 완료 | AI 피드백 생성·검토 |
| 🟢 연두 | `LIGHT_GREEN` | 피드백 받고 **PRD 개선 재제출** 완료 | 링크 개선 독려 |
| 🟢 초록 | `GREEN` | 피드백 받고 **PRD+링크 둘 다 개선** 완료 | 3차 심화 피드백 제공 |

**상태 전이 규칙 (반드시 이 로직대로):**
```
초기:                                   RED
PRD만 제출:                             ORANGE
PRD + 링크 둘 다 제출:                  YELLOW
(강사 피드백 발송 후)
  → PRD 개선 재제출:                    LIGHT_GREEN
  → PRD + 링크 둘 다 개선 재제출:        GREEN
GREEN 도달 시:                          3차 심화 피드백 버튼 활성화
```
- 색상 판정은 제출 상태에서 **자동 계산**(수동 변경 금지).
- 강사는 빨강·주황을 중심으로 순회하며 멘토링, 초록은 3차 피드백으로 연결.

## 4. 데이터 모델 — Firebase (Firestore)

**Firestore**를 사용한다. 실시간 구독(`onSnapshot`)을 지원하므로 5-6의 실시간 대시보드 갱신에 그대로 활용한다. 컬렉션 2개로 구성.

```
컬렉션  participants  (문서 ID = 자동 생성)
  - school              (학교명, 직접 입력)
  - name                (이름, 직접 입력)
  - prdTextV1           (1차 PRD 본문)
  - prdTextV2           (개선 PRD 본문, 없으면 null)
  - artifactUrlV1       (1차 산출물 링크)
  - artifactUrlV2       (개선 산출물 링크, 없으면 null)
  - status              (RED/ORANGE/YELLOW/LIGHT_GREEN/GREEN — 자동계산)
  - createdAt / updatedAt   (serverTimestamp)

컬렉션  feedbacks  (문서 ID = 자동 생성)
  - participantId       (participants 문서 참조)
  - round               (1 = 1차, 3 = 심화)
  - aiDraft             (AI가 생성한 초안)
  - instructorFinal     (강사가 검토·수정한 최종본, 없으면 null)
  - reviewed            (강사 검토 완료 여부 boolean)
  - linkTestLog         (링크 테스트 결과, map 또는 문자열)
  - createdAt / sentAt
```

**식별 규칙**: 제출 시 `school` + `name`으로 participants를 쿼리해서, 있으면 그 문서를 업데이트, 없으면 새 문서 생성(중복 카드 방지).

**Firebase 설정 메모 (Claude Code 참고)**
- Firebase 프로젝트 생성 → 웹 앱 추가 → `firebaseConfig` 객체를 앱에 넣어 초기화(`initializeApp`).
- Firestore 활성화. 초기엔 test 모드로 빠르게 시작하되, **배포 전 보안 규칙 필수**(아래 8장).
- 실시간 대시보드는 `onSnapshot`으로 participants 컬렉션을 구독 → 색 변화 자동 반영.
- `firebaseConfig`의 apiKey 등은 코드에 들어가지만, 이는 식별자 성격. 실제 보호는 **Firestore 보안 규칙**으로 한다(8장).

## 5. 기능 명세

### 5-1. 수강생 — 제출 화면
- 학교명·이름 입력(드롭다운 또는 텍스트) → 본인 카드 식별/생성.
- **PRD 제출**: 텍스트 영역에 PRD 붙여넣기(교육 설계+기술 설계 포함). 저장 시 상태 자동 갱신.
- **산출물 링크 제출**: 배포 URL 입력란.
- **피드백 확인**: 강사가 최종 발송한 피드백을 읽기 전용으로 표시(AI 초안이 아니라 강사 검토본만 노출).
- **개선 재제출**: 피드백 확인 후 PRD_v2 / 링크_v2 입력란이 열림. 각각 제출하면 연두→초록으로 전이.

### 5-2. AI 피드백 생성 (PRD)
- 입력: 수강생의 PRD 본문.
- 출력: **잘한 점 + 개선 사항**을 세부적으로. 그리고 **교육적 관점 피드백**을 별도 섹션으로 반드시 포함.
  - 기능/구조 관점: PRD 구성요소 충실도, 기능 정의 명확성, 데이터·AI 설계 타당성.
  - **교육적 관점**: 대상 학생 특성 반영도, 학습 목표의 적절성, 개별화·성공 경험 설계, 특수교육 적합성, 접근성(큰 글자·타이핑 최소화 등).
- 출력은 `ai_draft`에 저장하고 **곧바로 발송하지 않는다**(아래 검토 게이트 참조).

### 5-3. AI 피드백 생성 (산출물 링크) — 2단 검증
링크가 들어오면 다음 순서로 검증 후 피드백:
1. **1차 — HTML 파싱**: 링크의 HTML을 가져와 구조·텍스트·버튼·입력요소를 분석. 가능하면 이걸로 피드백.
2. **2차 — 브라우저 제어(폴백/심화)**: 파싱만으로 부족하거나 동적 앱이면, **크롬 브라우저를 자동 제어**해 실제 앱을 실행 → 버튼 클릭·타이핑 등 상호작용을 시도 → 동작 여부를 확인하고 그 결과로 피드백.
   - 테스트 로그(무엇을 눌렀고 어떻게 반응했는지)를 `link_test_log`에 남겨 강사가 검토 시 참고.
   - 브라우저 자동화는 Playwright(권장) 또는 Puppeteer 등 Claude Code가 적절히 선택.
- 여기서도 **교육적 관점**(학생이 실제로 쓰기 쉬운가, 성공 경험을 주는가)을 피드백에 포함.

### 5-4. 강사 검토 게이트 (중요)
- AI가 피드백 초안을 만들면 **강사에게 알림**(대시보드 배지 + 검토 대기 목록).
- 강사는 초안을 열어 **수정/가필**한 뒤 `instructor_final`로 저장 → **[최종 발송]** 버튼을 눌러야 수강생에게 노출.
- **AI 초안은 절대 수강생에게 자동 노출되지 않는다.** 강사 검토·발송을 거친 것만 보인다.

### 5-5. 3차 심화 피드백 (초록 도달자)
- 상태가 `GREEN`이 되면 **3차 피드백 버튼**이 활성화.
- 3차는 평가가 아니라 **앱에 기능을 추가·개선하는 방향**의 제안 중심(예: "여기에 음성 안내를 더하면", "학생별 난이도 분기를 추가하면").
- 동일하게 AI 초안 → 강사 검토 → 발송 흐름을 따른다.

### 5-6. 강사 대시보드 (방사형 + 실시간)
- **방사형(radial) 레이아웃**: 중앙에 강사, 둘레에 수강생 노드를 배치. 각 노드는 신호등 색상 원.
- **실시간 갱신**: 누군가 제출/개선하면 색이 즉시 바뀜(폴링 또는 실시간 구독).
- 노드 클릭 → 해당 수강생 상세(제출물·피드백 상태·링크 테스트 로그).
- **요약 위젯**: 색상별 인원수(빨강 n, 주황 n, 노랑 n, 연두 n, 초록 n), 검토 대기 피드백 수.
- 색상 필터: 빨강·주황만 보기(멘토링 모드), 초록만 보기(3차 피드백 모드).
- 디자인은 연수 시리즈와 통일감 있게 **귀여운 파스텔 톤 + 큰 신호등 원**. 색약 대비를 위해 색상 + 아이콘/숫자 병기.

## 6. 화면 목록 (라우트)

| 경로 | 화면 | 대상 |
|---|---|---|
| `/` | 수강생 제출/확인 (학교·이름으로 진입) | 수강생 |
| `/instructor` | 방사형 대시보드 (비밀번호 보호) | 강사 |
| `/instructor/review/:id` | AI 초안 검토·수정·발송 | 강사 |
| `/instructor/participant/:id` | 수강생 상세 + 링크 테스트 로그 | 강사 |

## 7. 구현 순서 (Claude Code 단계별)

1. **Firebase 연결 + 데이터 모델 + 상태 계산 로직**: Firebase 프로젝트 연결(firebaseConfig·Firestore 초기화), participants/feedbacks 컬렉션, 5단계 신호등 자동 판정 함수부터. 상태 전이 규칙을 함수로 만들고 검증.
2. **수강생 제출 화면**: 학교·이름 식별, PRD/링크 1차 제출, 상태 갱신 확인.
3. **강사 방사형 대시보드(읽기)**: 노드·색상·요약 위젯. Firestore `onSnapshot` 구독으로 실시간 갱신.
4. **AI 피드백(PRD)**: 잘한 점/개선/교육적 관점 3블록 생성 → ai_draft 저장.
5. **강사 검토 게이트**: 알림 → 검토·수정 → 최종 발송 → 수강생 노출.
6. **수강생 개선 재제출**: PRD_v2/링크_v2 → 연두/초록 전이.
7. **링크 검증 1단 (HTML 파싱)**: 링크 가져와 구조 분석 후 피드백에 반영.
8. **링크 검증 2단 (브라우저 제어)**: Playwright로 실제 클릭·타이핑 테스트 + 로그.
9. **3차 심화 피드백**: 초록 도달자 대상 기능 추가·개선 제안.

> 1~3단계까지만 해도 "제출 + 신호등 대시보드"가 돌아간다. 4단계부터 AI, 7~8단계가 링크 검증, 9단계가 심화. 시간이 부족하면 7·8을 뒤로 미뤄도 전체가 동작한다.

## 8. 보안·윤리 (반드시 반영)

- **개인정보 최소화**: 수강생이 PRD에 적는 학생 정보는 **별칭·번호만**. 실명·사진·민감정보 입력을 막거나 경고.
- **Firestore 보안 규칙(필수)**: test 모드로 시작하되 배포 전 규칙을 반드시 설정. 강사 전용 작업(피드백 발송·전체 조회)과 수강생 작업(본인 제출)을 구분. `firebaseConfig`는 코드에 노출되어도 무방하나, 실제 데이터 보호는 보안 규칙으로 한다. 민감정보가 없도록 별칭·번호 원칙과 함께 운영.
- **AI 키**: 피드백 생성에 쓰는 LLM API 키는 **서버 측 비밀(secret/환경변수)**로. 클라이언트 코드·화면에 노출 금지. (Firestore의 firebaseConfig와는 별개 — LLM 키는 절대 클라이언트에 두지 않는다.)
- **강사 화면 보호**: `/instructor`는 비밀번호 또는 접근 제어.
- **링크 테스트 안전장치**: 브라우저 자동화는 제출된 URL만 대상으로, 타임아웃·예외 처리. 외부 사이트 무한 탐색 금지.
- **AI 피드백 책임**: AI 초안은 반드시 강사 검토를 거쳐 발송(5-4). AI가 단독으로 학생에게 피드백하지 않는다.

## 9. 범위 밖 (이번엔 안 함)

- 정식 회원가입/SSO, 권한 등급 세분화
- 점수·등수 매기기(경쟁 유발 지양 — 색은 진도 표시용일 뿐)
- 모바일 앱(웹 반응형으로 충분)
- 자동 채점·합불 판정
